What are git and github ?

Git :

Git is a free and open-source distributed "version control system". It helps developers track changes in their code and collaborate with others. It was created by Linus Torvalds in 2005.

Version Control System :

A version control system (VCS) is a tool that helps track and manage changes to files over time. It allows you to save different versions of your project so you can go back to a previous version if something goes wrong. This is especially useful when working on large projects or with a team.

GitHub :

GitHub is a cloud-based platform that allows you to store, manage, and share your Git repositories. It adds collaboration features like pull requests, issue tracking, and more.

Why are Git and GitHub important ?

  1. Tracks Changes Over Time

    2. A VCS records every change made to files, allowing you to see what was modified, when, and by whom. This helps in understanding the evolution of a project.

  2. Restores Previous Versions

    3. If a mistake is made or if something breaks, a VCS lets you easily roll back to an earlier, stable version of the file or project.

  3. Keeps a History Log

    4. Every change is logged with a message, timestamp, and author name, helping you understand the reasoning behind changes and keeping a record for future reference.

  4. Supports Branching and Experimentation

    5. Developers can create separate branches to test features or ideas without affecting the main project. Once tested, changes can be merged back safely.

  5. Useful for Both Code and Content

    6. Although commonly used for code, version control can also be used for documents, designs, or any files where changes matter.
